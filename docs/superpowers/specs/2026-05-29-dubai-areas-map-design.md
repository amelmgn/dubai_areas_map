# Интерактивная карта районов Дубая — дизайн

**Дата:** 2026-05-29
**Статус:** утверждён

## Цель

Публичный сайт с интерактивной картой Дубая, показывающий три уровня административных
зон (секторы → районы → коммьюнити). При наведении на полигон подсвечивается заливка,
контур и название. Уровни сменяют друг друга по мере приближения карты. Сайт —
статический, разворачивается на Netlify.

## Исходные данные

В папке `geojson/` лежат три файла:

| Файл | Объектов | Свойства | Уровень |
|------|----------|----------|---------|
| `dubai_sectors.geojson` | 9 | `name`, `slug`, `lat`, `lon`, `kind` | секторы |
| `dubai_districts.geojson` | 14 | `name`, `kind` | районы (districts) |
| `dubai_locations.geojson` | 227 | `name`, `kind` | коммьюнити |

Все геометрии — полигоны. У секторов есть готовые координаты центра (`lat`/`lon`);
у районов и коммьюнити центр для подписи вычисляется.

## Технические решения

- **Движок карты:** MapLibre GL JS (open source, без токенов и лимитов). GeoJSON
  грузятся напрямую в браузер как источники (суммарно ~248 КБ).
- **Подложка:** растровые тайлы Carto Positron (светлая) по умолчанию, с переключателем
  на Carto Dark Matter (тёмная). Бесплатно, требуется атрибуция Carto + OpenStreetMap.
- **Без бэкенда:** чистая статика.
- **Окраска:** заливка полигона определяется его **сектором** (9 цветов палитры).
  Принадлежность к сектору вычисляется на этапе сборки (см. предобработку).

## Предобработка данных (`scripts/enrich.mjs`)

Node-скрипт на `@turf/turf`, запускается через `npm run build`. Делает следующее:

1. Для каждого района и коммьюнити определяет родительский сектор: проверяет, в какой
   из 9 секторных полигонов попадает центроид объекта (`turf.centroid` +
   `turf.booleanPointInPolygon`). Записывает `properties.sector` (значение `slug`
   сектора, напр. `"sector-1"`). Если центроид не попал ни в один сектор — fallback
   на сектор с максимальной площадью пересечения (`turf.intersect` + площадь); если и
   так пусто — `properties.sector = null` (полигон красится нейтральным серым).
2. Для каждого объекта всех трёх уровней вычисляет точку подписи: у секторов берётся
   готовая `lat`/`lon`, у остальных — `turf.centroid` (при необходимости
   `turf.pointOnFeature`, чтобы точка гарантированно лежала внутри полигона).
   Сохраняется как `properties.labelLng` / `properties.labelLat`.
3. Пишет обогащённые файлы в `public/data/` (`sectors.geojson`, `districts.geojson`,
   `communities.geojson`).

Скрипт идемпотентен: повторный запуск даёт тот же результат. Исходные файлы в
`geojson/` не меняются.

## Слои карты и поведение зума

Уровни **сменяют** друг друга (в каждый момент виден один уровень):

| Уровень | Видимость (zoom) |
|---------|------------------|
| Секторы | `zoom < 11` |
| Районы | `11 ≤ zoom < 13` |
| Коммьюнити | `zoom ≥ 13` |

- Видимость задаётся через `minzoom`/`maxzoom` слоёв.
- На границах диапазонов заливка плавно затухает/появляется (интерполяция
  `fill-opacity` по `zoom`), чтобы переход не был резким.
- Пороги (11, 13) — стартовые значения, подбираются по факту на реальной карте.

Для каждого уровня создаётся набор слоёв:

- **fill** — полупрозрачная заливка, цвет по `sector` (выражение
  `match`/`["get","sector"]` → цвет из палитры). Базовая непрозрачность ~0.15–0.2.
- **outline** (line) — тонкий контур по умолчанию.
- **label** (symbol) — подпись `name`, размещается в точке `labelLng/labelLat`.
  По умолчанию подписи **едва видны** (низкая непрозрачность текста, стартовое
  значение ~0.25 — подбирается по факту, чтобы не было каши при 227 коммьюнити).
  У наведённого объекта подпись становится обычной яркой (полная непрозрачность,
  через `feature-state hover`). При перегруженности можно дополнительно включить
  встроенное прореживание подписей MapLibre (по умолчанию они не накладываются).

## Взаимодействие

- **Наведение (hover):** через `feature-state` (`hover: true`). У наведённого полигона:
  заливка насыщается (выше `fill-opacity`, более плотный цвет), контур становится толще
  и ярче, а его подпись из «едва видимой» становится яркой (полная непрозрачность).
  Реализация на событиях `mousemove`/`mouseleave` по слою заливки текущего уровня. На тач-устройствах hover не работает — там роль
  «наведения» выполняет тап (показ названия + подсветка) без зума, отдельным
  лёгким поведением.
- **Клик:** `map.fitBounds()` к bbox полигона (`turf.bbox`) с отступами и анимацией.
  Приближение само переключает уровень вглубь.
- **Поиск:** текстовое поле с автодополнением по `name` всех трёх уровней (секторы,
  районы, коммьюнити). Список строится один раз из загруженных данных. Выбор пункта →
  `fitBounds`/`flyTo` к объекту + временная подсветка (`feature-state selected`).
- **Переключатель темы:** кнопка светлая ⇄ тёмная. Меняет источник тайлов подложки и
  палитру обводок/подписей; цвета секторов адаптируются (более яркие на тёмной).
- **Легенда:** компактный блок со списком 9 секторов и их цветами; на тёмной теме
  цвета синхронизируются.

## Структура проекта

```
dubai_areas_map/
├── geojson/                       # исходники (read-only, не меняются)
│   ├── dubai_sectors.geojson
│   ├── dubai_districts.geojson
│   └── dubai_locations.geojson
├── scripts/
│   └── enrich.mjs                 # предобработка (Turf)
├── public/                        # ← Netlify публикует эту папку
│   ├── index.html                 # разметка: контейнер карты, поиск, легенда, тема
│   ├── style.css                  # стили UI
│   ├── app.js                     # инициализация MapLibre, слои, события
│   └── data/                      # вывод enrich.mjs (обогащённые geojson)
│       ├── sectors.geojson
│       ├── districts.geojson
│       └── communities.geojson
├── package.json                   # devDep: @turf/turf; "build": "node scripts/enrich.mjs"
└── netlify.toml                   # build="npm run build", publish="public"
```

MapLibre GL JS и его CSS подключаются с CDN (или вендорятся локально в `public/`).
Палитра секторов и пороги зума выносятся в константы в начале `app.js` для лёгкой
настройки.

## Деплой (Netlify)

- `netlify.toml`:
  ```toml
  [build]
    command = "npm run build"
    publish = "public"
  ```
- `npm run build` генерирует `public/data/` из `geojson/` через `enrich.mjs`.
- Подключение Git-репозитория к Netlify → автоматическая сборка и раздача.
  Сайт доступен по URL Netlify; при желании — кастомный домен.
- Поскольку `public/data/` генерируется сборкой, его можно не коммитить
  (добавить в `.gitignore`), полагаясь на build step.

## Границы (не входит)

- Описания/детальные карточки объектов (в данных только `name`) — панель деталей не
  делаем; клик только зумит. Можно добавить позже, если появятся данные.
- Редактирование данных через UI.
- Серверная часть, аналитика, авторизация.

## Открытые мелочи (решаются при реализации)

- Точные пороги зума и параметры затухания — подбор на реальной карте.
- Палитра из 9 цветов для секторов (контрастная, спокойная) — подбирается при вёрстке.
- Поведение объектов без определённого сектора (`sector = null`) — нейтральный серый.
