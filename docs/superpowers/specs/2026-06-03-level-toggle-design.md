# Переключатель уровней (секторы / дистрикты / локации)

Дата: 2026-06-03

## Проблема

Сейчас три уровня данных (sectors / districts / communities) жёстко привязаны
к диапазонам зума через `minzoom`/`maxzoom` (`LEVELS` в `public/app.js`). На каждом
зуме виден ровно один уровень, и переход между ними происходит автоматически при
масштабировании. Это неудачное решение: пользователь не может, например,
рассматривать секторы вблизи или локации на общем плане.

## Цель

Заменить автоматическую привязку к зуму на ручной переключатель с тремя режимами:
**Секторы**, **Дистрикты**, **Локации** (communities). При выборе режима
соответствующая сущность отображается на всех зумах, пока пользователь не
переключит режим вручную.

## Поведение

- Зум-бэндинг убирается полностью. Все три уровня получают полный диапазон зума
  (`minzoom: 0`, `maxzoom: 24`).
- Видимость уровня задаётся вручную переключателем через `layout.visibility`
  (`'visible'` / `'none'`) на всех четырёх слоях уровня: fill, line, label,
  label-hover. Слои не пересоздаются.
- В каждый момент виден ровно один уровень — выбранный.
- Режим по умолчанию при загрузке — **Локации** (`communities`).

## Состояние

- Новая переменная модуля `activeLevel` (строка-id уровня, по умолчанию
  `'communities'`).

## Функция `setActiveLevel(id)`

1. Для каждого из трёх уровней выставляет `visibility` его четырёх слоёв:
   `'visible'` если `lvl.id === id`, иначе `'none'`.
2. Вызывает `clearHover()` — чтобы подсветка/hover-подпись не «зависала» на слое,
   который стал невидимым.
3. Обновляет активную кнопку сегмент-контрола (CSS-класс `active`).
4. Сохраняет `activeLevel = id`.

## UI

В `public/index.html` добавляется сегмент-контрол сверху по центру над картой:

```
┌─────────┬───────────┬──────────┐
│ Секторы │ Дистрикты │ Локации  │
└─────────┴───────────┴──────────┘
```

- Три кнопки с `data`-атрибутом id уровня (`sectors` / `districts` / `communities`)
  и подписями «Секторы» / «Дистрикты» / «Локации».
- Активная кнопка визуально выделена (класс `active`).
- Стили в `public/style.css` под существующие светлую и тёмную темы (по образцу
  имеющихся контролов — `#theme-toggle`, `#legend`).
- Обработчики кликов вешаются один раз в `init()` (не на каждый `style.load`),
  по образцу того, как `eventsWired` защищает hover/click.

## Интеграция со `style.load`

`addDataLayers()` вызывается на каждый `style.load`, включая смену темы. После
добавления слоёв он должен применить текущий `activeLevel` через
`setActiveLevel(activeLevel)`, чтобы выбранный режим сохранялся после переключения
темы (слои пересоздаются, дефолтная видимость у них `'visible'`).

## Поиск

Поиск продолжает искать по всем трём уровням. При выборе результата из поиска
(`wireSearch`), помимо `flyToBbox` и `setHover`, выполняется **автопереключение**:
вызывается `setActiveLevel(result.source)`, чтобы уровень выбранного результата
стал активным и подсветка легла на видимый слой. (Вариант «а» из обсуждения.)

`result.source` уже совпадает с id уровня (`sectors` / `districts` /
`communities`), так что отдельного маппинга не нужно.

## Индикатор зума

Текущий `updateZoomIndicator` вычисляет «уровень» из зум-порогов
(`LEVELS.find(l => z >= l.minzoom && z < l.maxzoom)`) — после удаления бэндинга это
теряет смысл. Заменяется на показ зума и активного режима, например
`z 12.34 · Локации`. Понадобится отображаемое имя уровня (короткая мапа
id → подпись, та же, что в кнопках).

## Без изменений

- `public/helpers.js` (палитра, поиск) — не трогаем.
- Легенда «Секторы» остаётся: заливка окрашена по свойству `sector` во всех
  режимах, легенда релевантна всегда.
- `scripts/enrich.mjs` и source-данные — не трогаем.
- Механика hover/highlight (`feature-state`, `setHover`/`clearHover`) — не трогаем,
  кроме вызова `clearHover()` при смене режима.

## Тестирование

- `npm test` (юнит-тесты `enrich`/`helpers`) должен оставаться зелёным — логика
  изменений в `helpers.js` не затрагивается.
- Ручная проверка в браузере (`npm run build && npm run serve`):
  - на старте видны Локации на любом зуме;
  - переключение между тремя режимами меняет видимый уровень и активную кнопку;
  - смена темы сохраняет выбранный режим;
  - выбор результата поиска другого уровня переключает режим и подсвечивает объект;
  - индикатор зума показывает активный режим.
